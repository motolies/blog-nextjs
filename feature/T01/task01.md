# 요구사항 

- `/api/jira/admin/sprint/{year}` 를 호출하면 아래와 같은 응답을 받을 수 있음 
- 연도를 넣으면 해당 연도의 주(week)를 2로 나누어서 sprint 라는 단위를 생성함 
- 스프린트 단위로 보고서를 작성하려고 함
- 차트 라이브러리는 `Apache ECharts` 를 사용하고 싶어.
- 막대 차트의 스프린트의 작업자 파트를 누르면 해당 작업자가 처리한 이슈를 별도로 호출하여 뿌리고 싶어.
  - api 별도 제공 예정 
- admin 페이지에서 헤더에 sprint 를 만들고 클락하면 페이지 입장 (/admin/sprint) 
- 기본적으로 올해 연도를 보여주고, 과거 3년까지 선택할 수 있는 셀렉트 박스를 만들어서 연도 선택시 해당 연도의 데이터를 호출하여 보여줌


### 보고서 페이지 간단 레이아웃
[년도 셀렉트(과거 3년부터 올해년도까지 선택 가능), 년도 선택시 조회]
<br>
[막대차트(막대 차트의 x 스프린트, y 스토리포인트 합계(세부로는 작업자))]  
[표(작업자별 스프린트별 스토리포인트 합계)]
<br>
[스프린트 표]

|   작업자   | 2024-SP-01 | 2024-SP-02 | 2024-SP-03 | ... | 2024-SP-26 | 계         |
|:-------:|:----------:|:----------:|:----------:|:---:|:----------:|-----------|
|  작업자 A  |   12.85    |    8.0     |    8.8     | ... |    11.2    | sum(작업자A) |
|  작업자 B  |    4.5     |    8.5     |    10.8    | ... |    12.2    | sum(작업자B) |
|  작업자 C  |    8.95    |    9.1     |    12.8    | ... |    13.2    | sum(작업자C) |
| 스프린트 합계 |   sum(계)   |   sum(계)   |   sum(계)   | ... |   sum(계)   | sum(계)    |
